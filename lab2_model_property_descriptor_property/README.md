# Лабораторная работа №2: Модель задачи: дескрипторы и @property

### Описание проекта
Во 2 лабе мы сделали систему надежной. Если в первой лабе мы просто собирали данные, то теперь мы добавили «фильтры». Теперь нельзя случайно или специально сломать задачу (например, поставить ей отрицательный приоритет или изменить её ID).

**Ключевые изменения:**
* **Валидация через Дескрипторы:** Поля `priority` и `status` защищены классами-дескрипторами.
* **Инкапсуляция:** Приватные атрибуты (`_id`, `_created`) скрыты от прямого изменения.
* **Computed Properties:** Свойство `is_urgent` вычисляется динамически на основе приоритета.
* **Read-only Data:** Идентификатор задачи и дата создания доступны только для чтения.

### Что было изучено
* **Протокол дескрипторов:** Реализация методов `__get__`, `__set__` и `__set_name__` для создания переиспользуемой логики валидации.
* **Декоратор @property:** Создание управляемых атрибутов (геттеров) для реализации защиты Read-only.
* **Инкапсуляция:** Использование защищенных атрибутов для разграничения интерфейса и внутреннего состояния объекта.

### Установка и запуск

**Локальная установка зависимостей:**
```bash
pip install -r requirements.txt
```

**Запуск основной программы:**
```bash
python3 -m src.main
```

**Запуск тестов и проверка покрытия:**
```bash
pytest --cov=src tests/
```

### Работа в Docker


**1. Сборка образа:**
```bash
docker build -t lab2-tasks .
```

**2. Запуск тестов и отчета о покрытии**
```bash
docker run --rm lab2-tasks
```

**3. Запуск самой программы внутри контейнера:**
```bash
docker run --rm lab2-tasks python -m src.main
```
